Module 2 · Financial‑Data Sourcing & SQL

Download two price series · Import with pgAdmin · First JOIN query (no coding skills needed)

0 · Why do we care?

Before we forecast or back‑test anything, we need clean financial price data. This mini‑lab walks you through grabbing equity, FX, and crypto time‑series in CSV format and parking them in a relational database without writing code.

1 · One‑Click Setup

  1. PostgreSQL 16 + pgAdmin 4 — download the bundled installer (Windows / macOS / Linux). Accept all defaults and remember the admin password.
  2. Launch pgAdmin and confirm you see “PostgreSQL 16” in the tree.

3 · Import CSVs into PostgreSQL (GUI‑only)

Goal: two tables called spy_daily and eurusd_daily.

Show 4‑step wizard ▾
  1. Right‑click Databases ▸ postgres ▸ Schemas ▸ public → Create › Table…
  2. Name table spy_daily. In Columns tab add date (DATE) & close (NUMERIC).
  3. After saving, right‑click table → Import/Export. Format CSV, select spy_daily.csv, Delimiter ,, Header ✓.
  4. Repeat for eurusd_daily (columns: date, close).
-- OR go directly to the Query section
CREATE TABLE eurusd_daily (
    date     DATE,
    open     NUMERIC,
    high     NUMERIC,
    low      NUMERIC,
    close    NUMERIC
);

COPY eurusd_daily
FROM 'D:/Downloads/eurusd_d.csv'   -- adjust path
WITH (FORMAT csv, HEADER, DELIMITER ',');
-- Repeat for spy_daily

4 · First Peek at SQL (Copy‑paste)

Open the Query Tool in pgAdmin and paste:

-- view recent SPY closes
SELECT * FROM spy_daily ORDER BY date DESC LIMIT 5;

-- merge SPY & EURUSD on date to check co‑movement
SELECT s.date, s.close AS spy_close, e.close AS eurusd_close
FROM spy_daily s
JOIN eurusd_daily e ON s.date = e.date
ORDER BY s.date DESC
LIMIT 5;

You just joined two asset classes on a common key (date) without writing any Python.
